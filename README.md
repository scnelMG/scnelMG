## 한 줄 소개

데이터 분석과 머신러닝 프로젝트를 중심으로 포트폴리오를 정리하고 있습니다. 단순히 모델을 학습시키는 것보다 **문제 정의, 데이터 공개 경계, 모델링 의사결정, 결과 근거, 재현 가능성, 한계**가 함께 보이는 저장소를 지향합니다.

관심 분야는 tabular ML, 추천/랭킹, NLP, vision-language model, 공공데이터 서비스, 금융/보험 데이터 분석입니다.

## 프로젝트를 정리한 기준

- **문제 정의**: 모델링 전에 어떤 의사결정을 돕는 프로젝트인지 명확히 씁니다.
- **내 역할**: 팀 프로젝트는 단독 제작처럼 보이지 않게 기여 범위를 분리합니다.
- **기술적 의사결정**: 모델, 검증, feature engineering 선택 이유를 설명합니다.
- **결과 근거**: leaderboard나 단일 점수보다 pipeline, validation, feature evidence를 함께 남깁니다.
- **재현 가능성**: 공개 checkout에서 가능한 것과 원본 데이터가 필요한 것을 구분합니다.
- **공개 안전성**: 원본 데이터, 비공개 자료, 대용량 artifact, 개인정보 가능 자료를 제외하고 문서화합니다.
